1. What Exactly is Caching? – Simple & Technical Definitions

00:00:00.240what exactly is caching to simplify it in one sentence caching is a mechanism using which we decrease the amount of time and effort it takes to perform some amount of work that is the on line explanation of what exactly is caching and a more technical definition of that would be caching is basically keeping a subset of some data let's say we have a primary data and when we keep a subset not the whole data a subset of that data depending on the uses of the data the frequency of the uses and the probability of the next uses time etc etc depending on a lot of parameters when you keep a subset of some data in a location which is faster to access which takes less time and also it is easier which means it is it also takes less effort so technically caching is a mechanism using which we can decrease the amount of time and the effort it takes to retrieve or to do some kind of operation

Caching in One Sentence (Simple)

  • Mechanism to reduce time + effort for repeated work

Technical Definition

  • Store subset of primary data in faster-access location
  • Based on frequency, probability, recency of use
  • Key factor in high-performance apps (latency in ms/μs)

Analogy: Kitchen cache = frequently used spices on counter (fast access) vs full pantry (slower). Don't store everything — only what's used often.

2. Why Caching is Critical for High-Performance Apps

00:01:18.360this single mechanism or this single technique is a huge factor in a lot of high performance applications so let's go through a couple of examples and understand where exactly people use caching and how does it exactly impact the performance of the application and how important it is and to give you a hint it is very important at least when it comes to high performance applications which track latency in two digit micros or milliseconds

Why Cache? – Benefits Explained

  • Reduces latency (faster responses)
  • Saves compute resources (less CPU/memory)
  • Handles scale (millions of requests without overload)
  • Improves user experience (snappy apps)
  • Cost-effective (less DB/CPU usage = lower cloud bills)

Example 2: CPU Cache – Hardware Level Caching

Example 3: DNS Cache – Domain Resolution

00:06:00 approxlet's take the example of DNS cache when you type a domain name in your browser let's say you type google.com and you hit enter the browser first checks the local DNS cache to see if it has the IP address of google.com if it does it uses that IP address to connect to the server if it does not it sends a request to the DNS server to get the IP address and then it stores that IP address in the local DNS cache for future use so that the next time you type google.com the browser can directly use the IP address from the cache without having to send a request to the DNS server

5. Redis – Popular In-Memory Cache

01:02:20.000whenever we are talking about implementing rate limiting we make use of inmemory databases like redis instead of storing it in our relational databases

Redis Features Explained

  • In-memory → super fast (RAM access)
  • Key-Value store → simple get/set
  • Distributed → scale across machines
  • Persistence optional → AOF/RDB snapshots
  • Structures: strings, hashes, lists, sets, sorted sets
  • Pub/Sub, Lua scripting, expiration/TTL

Why Redis for Caching? Fast (sub-ms), easy eviction (LRU/LFU), high availability (replication).
