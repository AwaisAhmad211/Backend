# Backend

Learning structure:
- Daily notes live under notes/YYYY/MM/YYYY-MM-DD.md. Start with [notes/2026/01/2026-01-01](notes/2026/01/2026-01-01.md).
- Topic summaries go in [guides](guides/README.md) once you see repeated material.

Concept explorer (click a topic):
- Database → [guides/database.md](guides/database.md)
	- [Schema design](guides/database.md#schema-design)
	- [Indexes and performance](guides/database.md#indexes-and-performance)
	- [Transactions and isolation](guides/database.md#transactions-and-isolation)
- Authorization → [guides/authorization.md](guides/authorization.md)
	- [Role-based access control](guides/authorization.md#role-based-access-control)
	- [Token handling](guides/authorization.md#token-handling)
	- [Session management](guides/authorization.md#session-management)

How to add a new day:
- Copy the template from the latest note, adjust the date, and fill sections (Summary, Key Ideas, Code Snippets, Commands, References/TODO).

Navigation:
- Notes index: [notes/README.md](notes/README.md)
- Guides index: [guides/README.md](guides/README.md)

How to add a new concept:
- Add a section in the relevant guide and link the daily notes that explain it.
- If a new topic appears, create a guide file under guides and add it to the list above.
